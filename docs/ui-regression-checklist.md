# UI 手工回归清单（Linux.do / Notion / 通用网页）

> 适用脚本：`LinuxDo-Bookmarks-to-Notion.user.js`
>
> 建议每次 UI 大改后先跑静态校验：`node scripts/validate-userscript-ui.js`（或无 Node 时用 `python3 scripts/validate-userscript-ui.py`），再按本清单做手工回归。

## 0. 通用检查（所有站点）

- [ ] 页面刷新后 UI 样式不重复注入（无明显样式叠加/抖动/字体异常）
- [ ] 按钮/输入/选择器在 `Tab` 导航下有清晰焦点环（`:focus-visible`）
- [ ] 系统开启“减少动态效果”（`prefers-reduced-motion`）时，无明显闪烁/强动画

## 1. Linux.do（收藏页 UI 面板）

打开：`https://linux.do/u/<你的用户名>/activity/bookmarks`

- [ ] 面板出现、标题栏可拖拽移动；缩放手柄可用（左边/下边/左下角）
- [ ] “加载收藏列表”可用，计数更新，列表渲染正常
- [ ] 勾选/全选/取消全选正常，已选数量显示正确
- [ ] 导出流程按钮状态正确（开始/暂停/继续/取消），进度条与状态提示显示正常
- [ ] AI 对话区可发送、可清空；设置折叠可打开/关闭；输入框回车发送正常
- [ ] 日志面板可展开/收起，条目渲染正常；“清除日志”生效

## 2. Notion（浮动 AI 面板）

打开任意页面：`https://www.notion.so/`

- [ ] 右下角浮动按钮出现；可拖拽；点击可开合面板
- [ ] 面板可拉伸（左边/上边/左上角），关闭/展开逻辑正常
- [ ] 对话区可发送；快捷 chips 点击可触发发送
- [ ] “设置”折叠区可开合；保存设置后刷新仍能加载配置
- [ ] 状态提示（info/success/error）显示与关闭按钮正常

## 3. 通用网页（剪藏导出面板）

打开任意非 Linux.do/Notion 页面（例如新闻/博客/文档页）

- [ ] 右下角剪藏按钮出现；点击可打开/关闭面板
- [ ] 面板预览区域显示标题/来源等元信息
- [ ] 设置 API Key 的流程正常（Key 不预填到 DOM）
- [ ] 导出按钮可触发导出；exporting/success/error 三种状态视觉可区分且可读
- [ ] 面板可拉伸（左边/上边/左上角），不会遮挡主要操作按钮
