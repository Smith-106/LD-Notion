多源收藏导入

从 Linux.do、GitHub、浏览器书签、知乎与通用网页收集内容，统一写入 Notion 或导出到 Obsidian。

对话式 AI 工作区

在 Linux.do、Notion 和通用网页面板中用自然语言搜索、整理、创建、更新和总结内容。

双形态交付

支持 Tampermonkey 用户脚本，也支持独立 Chrome 扩展，按使用场景选择。

权限与审计守卫

写入入口统一经过 OperationGuard，按只读、标准、高级、管理员四级权限控制。

自动化导入与去重

Linux.do 与 GitHub 来源可独立配置自动导入、轮询间隔、去重策略和更新检查。

面向发布的验证链路

提供 baseline、扩展构建、bounded profile 和 UI 手工回归清单，降低交付回归风险。

推荐阅读路径 ​

  1. 先看 快速开始，确认你要使用脚本版还是独立扩展版。
  2. 按 Notion 配置 创建 Integration、授权并选择数据库或页面。
  3. 从 功能地图 选择你的入口：Linux.do 导出、GitHub 导入、浏览器书签、AI 助手或网页剪藏。
  4. 如果要二次开发，阅读 整体架构 与 开发与验证。

深入理解系统 ​

  • Concepts：从机制地图理解 LD-Notion 的核心边界。
  • Routing Rules：理解来源、目标、授权、AI 和权限如何共同决定处理路径。
  • Import Pipeline：理解内容从捕获到写入、审计和失败处理的状态流。
  • OperationGuard：理解写入动作前的权限、确认和审计网关。
  • Extension Architecture：理解用户脚本、扩展构建和部署边界。
